# PRODUCT REQUIREMENTS DOCUMENT (v1.2)

**Title:** Local-First Agent Runtime + Optional SaaS Control Plane

## Packaging Model (Critical)

The system uses a **single container** that contains two distinct components:

- **Agent Runtime** (the actual agent logic + harness)
- **Gateway** (handles ingress, egress, policy, and observability)

These two components are packaged together inside one container image and deployed as one atomic unit. They communicate internally but remain separate logical components.

## Visual Representation

In the architecture diagrams:
- A container is shown as a box.
- Inside that box are two separate nodes: "Agent Runtime" and "Gateway".
- They are connected to each other.
- All external communication (user calls, LLM calls, MCP, A2A) goes through the Gateway component.

## Data Flow

- User → Gateway (inside container)
- Gateway routes to Agent Runtime (inside same container)
- Agent Runtime sends outbound requests back through the Gateway (same container)
- Telemetry from both components goes to the local dashboard

This model ensures the gateway is always present and controls all traffic for the agent it is packaged with.
