AgentPaaS — PRD v3.0
ADVISOR EDITION · JUNE 2026

The Governed Runtime for AI-Built Agents

The wedge: Coding agents (Claude Code, Codex, Hermes, Cursor) generate working agents in minutes — but no security or platform team will let that code run unattended with credentials. AgentPaaS turns AI-generated agent code into a governed, auditable production workload in one command.

00 — What changed from v2.0
Persona fix: v2.0 sold a Docker/mTLS/egress-policy product to "non-developer ops users." The real buyer of this product is the platform/security engineer — the person currently saying "no" to agent deployment. Non-developers become users in Phase 3, not buyers in Phase 1.
Moat fix: Do not build a gateway. agentgateway (Linux Foundation, Rust, MCP/A2A/LLM-native) already exists. Consume it. Your IP = harness, identity, policy UX, audit trail, packaging pipeline.
Architecture fix: "Two components in one container" is a deploy-time detail, not a product principle. Single artifact for laptop mode; sidecar mode for K8s; same policy and audit everywhere.
01 — The Niche Problem (start here, expand later)

Specific source

Agents written by AI coding tools — an exploding, underserved supply. "Agents build agents" (your own tagline) means the bottleneck is no longer creation.

Specific blocker

Security sign-off. AI-written code + live credentials + autonomous outbound calls = automatic "no" from every security team today.

Specific buyer

Staff platform engineer (champion), VP Eng / CISO (economic buyer) at Series B–D software companies, 200–2,000 employees, with active agent experiments.

03 — Phases
P0

Months 0–3 · Validate, then demo

  • 15 customer conversations → 5 committed design partners (Workato network = unfair advantage)
  • Build ONLY the demo: agent pack + default-deny egress + audit log + minimal dashboard on agentgateway
  • 3-minute video: AI writes agent → packed → blocked from exfiltrating → signed audit export
  • Kill condition: 15 conversations don't yield 5 partners → wedge is wrong, stop and re-aim
P1

Months 3–9 · OSS launch (Apache-2.0)

  • Full local runtime: harness adapters (Python loop, LangGraph, CrewAI), identity, secrets, signed images, dashboard
  • Distribution wedge: "deploy" integrations INSIDE Claude Code / Codex / Hermes — nobody owns this channel yet
  • Gate to P2: 1,000 weekly-active runtimes OR 25 production deployments
P2

Months 9–18 · Team control plane (BSL, paid)

  • Fleet view, RBAC, SSO, policy distribution, alerting, private signed agent registry with approval workflow
  • Paywall line: individual = free forever; organization = paid. Never paywall single-machine security.
  • Founder-led sales to OSS installs showing 3+ dashboard users. SOC 2 Type I by month 12.
P3

Months 18+ · The platform the landing page promises

  • Non-developer experience: pick blessed agent from internal registry, fill form, run — no Docker visible
  • Eval gates ("Cognitive Audit" becomes real), A2A topologies, regulated mid-market compliance pack, $100k+ ACVs
  • Marketplace only after 50+ paying logos

06 — Risks

HIGHPlatform absorption

Docker/GitHub/Anthropic ships native "agent deploy." Mitigation: move now, stay neutral across all coding tools, go deeper on policy/audit than a platform vendor will. This risk defines the 12–24 month window.

HIGHExecution configuration

Roadmap needs 2–3 infra engineers for a year. Solo non-eng founder, employed, interviewing = the binding constraint. Options: technical cofounder, pre-seed on demo + partners, or scope P0 to AI-assisted weekend-buildable demo.

MED-HIGHCategory timing

Enterprise "production agent governance" budgets may lag 12 months. Mitigation: design partners surface truth early; OSS runtime is cheap to keep alive while market matures.

MEDOSS→paid conversion

Base rates <1–5% of orgs. Mitigation: org/individual paywall line; RBAC + audit retention as hooks enterprises cannot decline.

MEDSecurity product burden + standards churn

One CVE in a security product is existential → embed foundation gateway, pentest before GA, ship SBOMs. MCP/A2A churn → consume behind adapters, never fork.

LOWBrand

"AgentPaaS" is descriptive and undersells (you sell governance, not hosting). Keep the domain; consider a product name within it.
