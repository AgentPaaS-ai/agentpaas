## Delivered

### 1. Effective call deadline
- `ServiceInstance.LeaseDeadline time.Time` — wall-clock deadline set at Start; checked before dispatch in ManagedServiceResolver.
- `EffectiveCallDeadline()` helper computes min of multiple time.Time sources.
- `mcpCallContext` in harness continues to use B30 TimeEnvelope's `EffectiveOperationDeadlineMs` (min of stall timeout, lease remaining, active remaining) for caller-side deadlines.
- Managed resolver checks `inst.LeaseDeadline` before dispatch; rejects with `mcp_lease_expired` if past.

### 2. Request/response bounds
- `MaxRequestBytes = 256 KiB` — enforced in Router.CallTool (before dispatch) and ManagedServiceResolver.ResolveToolCall, and harness handleMCP.
- `MaxResponseBytes = 1 MiB` — enforced mirror of existing maxBodySize; checked via readLimitedHTTPResponseBody.
- `MaxJSONDepth = 32` — recursive json.Decoder walk; enforced on request input and response output in Router and ManagedServiceResolver.
- New error codes: `ErrCodeBodyTooLarge`, `ErrCodeDepthTooDeep`.

### 3. Concurrency
- `CallSemaphore` — bounded concurrent call gate; rejects immediately (no queue) when at capacity with `ErrCodeOverloaded`.
- Per-service concurrency via `Router.SetServiceConcurrency()` and `Router.getSemaphore()`.
- Per-caller concurrency via `harnessRPCServer.mcpSemaphore` with `DefaultMaxConcurrentMCPCalls = 8`.
- `MaxConcurrency` field on `ServiceInstance` for service-declared limits.

### 4. Cancellation
- `CancelTracker` — tracks in-flight call cancel funcs per ServiceInstance.
- `ServiceRegistry.Fence` cancels all tracked in-flight calls before transitioning to FENCED state.
- `RegisterCall` / `UnregisterCall` on ServiceRegistry for resolver-managed call lifecycle.
- Late results after cancel discarded (cancel func called, context cancelled).

### 5. Heartbeat cannot extend caller/workflow ceilings
- `TestHeartbeat_DoesNotExtendCallerCeiling` proves LeaseDeadline is immutable after set.
- `copyInstance` preserves LeaseDeadline exactly.

### 6. Audit
- All denial paths record reason + timing via `AuditToolDenied` with status codes: `request_too_large`, `request_too_deep`, `response_too_deep`, `overloaded`.
- Digests only (hashRouterJSON for input, RedactToolOutputHash for output) — no raw bodies.

### 7. No cross-service fallback
- Router fails closed if primary binding fails (`TestRouter_CallTool_NoCrossServiceFallback_FailsClosed`).
