## Delivered

### 1. Call + lifecycle evidence records
- `MCPCallRecord` struct: caller_run_id, caller_attempt_id, service_run_id, service_attempt_id, lease IDs, workflow_id, binding_id, tool, input_digest, output_digest, protocol_version, status (SUCCEEDED/FAILED/CANCELLED/UNKNOWN/OVERLOADED/TIMEOUT), reason (redacted), timing_ms, started_at, finished_at, evidence_refs.
- `MCPServiceLifecycleEvent` struct: correlation_id, workflow_id, service_binding_id, run_id, attempt_id, lease_id, generation, from_state, to_state, reason, timestamp.
- `CallEvidenceStore` interface with `InMemoryCallEvidenceStore` implementation.
- `InFlightCallTracker`: tracks in-flight calls for restart/fence marking as UNKNOWN/CANCELLED — never SUCCEEDED.

### 2. Wire recording
- `Router.CallTool` creates correlation ID at start, records UNKNOWN, then updates to SUCCEEDED/FAILED/OVERLOADED/TIMEOUT on completion. Single correlation_id per call mitigates T05 R2 double-audit.
- `Router.SetEvidenceStore(store)` installs the evidence store.
- `ServiceRegistry` emits lifecycle events on Declare, Start→Ready, Fence, Stop→Stopped, and failInstance transitions.
- `ServiceRegistry.SetEvidenceStore(store)` installs the evidence store.
- `Router.SetEvidenceStore` and `ServiceRegistry.SetEvidenceStore` both use separate instances — caller can wire the same store across both for unified evidence.

### 3. Health / status exposure
- `ServiceRegistry.HealthSummary(workflowID, serviceBindingID)` returns readiness, bounded failures (ring buffer, max 16), generation, lease deadline, active calls per service.
- `ServiceRegistry.RecordHealthFailure(workflowID, serviceBindingID, code, reason, tool)` records failures from Router typed errors.
- `WorkflowHealthSummary` aggregates health across all services in a workflow.

### 4. Restart semantics
- `CallEvidenceStore.MarkInFlightUnknown(workflowID)` marks all in-flight calls as UNKNOWN with restart reason.
- `InFlightCallTracker` provides snapshot of in-flight calls for fence/restart reconciliation.
- No auto-reissue: only the evidence is updated; caller must decide.
- Late result after fence/restart is discarded — call status is CANCELLED/UNKNOWN, never SUCCEEDED.

### 5. Cleanup idempotency
- `ServiceRegistry.CleanupServiceResources(workflowID, serviceBindingID)` removes container, network attachments, network if empty, clears capability/endpoint. Safe to call twice.
- `ServiceRegistry.DiscoverOrphans(workflowID)` finds containers not tracked by active instances (extends T03/T04 reconcile).
- `ServiceRegistry.Reconcile` already discovers and marks unhealthy containers; orphan discovery adds explicit listing.

### 6. Secret safety
- Evidence structs contain NO raw args/results, capability tokens, network aliases, credentials, or private keys.
- Tests scan serialized evidence JSON for forbidden substrings (sk-, AKIA, ghp_, -----BEGIN, PRIVATE KEY, xoxb-, etc.).
- Struct fields explicitly exclude input/output/capability/endpoint/alias/credential fields.
