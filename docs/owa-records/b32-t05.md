# B32-T05: Event-driven task wait/wake and terminal outbox

**Status:** COMPLETE
**Date:** 2026-07-21
**Commit:** 263c60a

## Deliverables

### task_outbox.go — TaskOutbox
Wraps delegation.Store. `CommitTerminal` commits a terminal task transition
(CAS), result (PutResult), and event (AppendEvent) with the same atomicity
contract as trigger.Outbox:
1. Validate transition (TaskStatus -> terminal EventType mapping)
2. CASTask (state commit)
3. PutResult (if not DENIED)
4. AppendEvent (outbox event)
Ordering: state-first, then event. If event append fails, state mutation
is treated as uncommitted (caller retries; idempotent CAS resolves).

### task_waiter.go — TaskWaiter
- `WaitTerminal(ctx, taskID, afterSeq)` — blocks until terminal, returns
  Task + all events > afterSeq. Subscribe-then-check prevents race.
- `Subscribe(ctx, taskID, afterSeq)` — returns buffered event channel + cancel;
  channel closes on terminal or context cancel.

### memory_store.go — subscriber support
- sync.Cond broadcast on AppendEvent wakes all waiters
- `eventSubscriber` with idempotent close via sync.Once
- Duplicate EventID detection (rejected with error)
- `terminalSignaled` per task for fast-isTerminal check
- `SubscribeEvents` replays events > afterSeq, registers subscriber,
  delivers non-blocking via buffered channel

### store.go — interface
Added: `SubscribeEvents(ctx, taskID, afterSeq) (<-chan TaskEvent, func(), error)`

## Design decisions
- TaskOutbox modeled after trigger.Outbox (same phase ordering)
- Waiter uses Subscribe-then-check pattern to prevent lost-wake race
- MemoryStore uses sync.Cond + buffered channels; no polling
- Terminal events auto-close subscriber channels (idempotent via sync.Once)
- Non-blocking fan-out: `select default` drops to slow consumers
