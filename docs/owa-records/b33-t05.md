# OWA Record — B33-T05 Wire Real MCP Protocol Router End to End

**Date:** 2026-07-23

## Delivered

### 1. Production router install
- `cmd/harness/main.go`: Always constructs and installs a `mcpmanager.Router` at startup via `server.SetRouter()`.
- `internal/harness/server.go`: Added `SetRouter()` method that propagates to the internal `harnessRPCServer`.
- The Router starts with an empty `Manager`; server registrations populate at runtime.
- When no servers are registered, MCP calls fail closed with "mcp server/tool not allowed" — no synthetic success.

### 2. AgentPaaS-service transport resolver
- `internal/mcpmanager/managed_resolver.go`: New `ManagedServiceResolver` backed by `ServiceRegistry`.
- `internal/mcpmanager/router.go`: Added `routeManagedService()` method that dispatches "agentpaas-service" transport through the resolver.
- Added `SetManagedResolver()` and `managedWorkflowID` fields to Router.
- Endpoint and capability are resolved from the registry — never from worker payload.

### 3. Context deadline propagation
- `internal/mcpmanager/router.go`: `decodeMCPResponse()` now derives timeout from `ctx.Deadline()` when available, falling back to fixed 5s `stdioResponseTimeout` for legacy callers.
- `internal/harness/rpc_server.go`: New `mcpCallContext()` creates a context with timeout derived from B30 `TimeEnvelope` when available (30s default fallback, documented as legacy/compat).
- `handleMCP()` passes this context to `router.CallTool()` instead of `context.Background()`.

### 4. Reject synthetic success for managed bindings
- `internal/harness/rpc_server.go`: `isManagedBinding()` detects bindings with transport "agentpaas-service".
- `handleMCP()`: Even with `AGENTPAAS_TEST_FAKE_MCP=1`, managed bindings get a typed denial (synthetic success forbidden).
- Non-managed (external stdio/HTTP) bindings still get synthetic success in test mode.

### 5. Typed errors
- `internal/mcpmanager/router.go`: New `TypedError` type with stable error code constants:
  - `ErrCodeProtocolError`, `ErrCodeServiceNotFound`, `ErrCodeServiceNotReady`
  - `ErrCodeLeaseExpired`, `ErrCodePolicyDenied`, `ErrCodeTimeout`
  - `ErrCodeCancelled`, `ErrCodeServiceCrashed`, `ErrCodeRouterUnavail`
- `internal/harness/rpc_server.go`: `mcpErrorCode()` maps both `TypedError` codes and legacy error strings to stable RPC response codes.

### 6. Characterization tests updated
- `TestCharacterization_MCPRouterNotInstalled`: Flipped to document router IS installed in production.
- `TestCharacterization_MCPManagerExistsButDaemonDoesNotUseIt`: Updated to permit daemon mcpmanager import; informational only.
- `TestCharacterization_MCPTimeoutInventory`: Documents context deadline propagation and 30s legacy fallback.
### 7. New tests (continued)

- `TestManagedServiceResolver_DistinctiveToolValue`: Unit test proves resolver hits correct endpoint from registry, sends Capability header, returns distinctive non-synthetic value (`source=b33-t05-managed-real`), rejects undeclared tools and not-ready instances.
- `TestManagedServiceResolver_NilRegistryFailsCloses`: Resolver fails closed when no registry is configured.
- `TestManagedServiceResolver_ContextDeadlineHonored`: Resolver propagates context deadline to HTTP call.
- `TestMCP_RouterDeliversManagedServiceResult`: Integration test proves full e2e path — Manager.Register with transport=agentpaas-service → Router → ManagedServiceResolver → httptest endpoint → distinctive value (`source=b33-t05-integration-real`, `value=managed-e2e-proof`). Also proves managed path cannot succeed without the resolver wired in.

### 8. Manager.Validate extended

- `Manager.Validate` now accepts transport `agentpaas-service` alongside `stdio` and `http`.

### 9. Test helpers exported

- `mcpmanager.TestServiceRegistry()`, `mcpmanager.TestServiceInstance()`, `mcpmanager.TestManagedResolverHTTPClient()` — exported for cross-package integration tests that need pre-built ServiceRegistry instances without real containers.
