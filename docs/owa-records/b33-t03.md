## Design decision: extend mcpmanager (not new package)

Chose to extend `internal/mcpmanager` with three new files rather than create a separate `internal/mcpservice` package. Reasoning:

1. `Manager` already owns the resource model (server IDs, allowed tools, policy digests).
2. `Lifecycle` already manages container start/stop/readiness via `RuntimeDriver`.
3. The existing `fakeRuntimeDriver` in lifecycle_test.go is shared by both old and new tests.
4. Coupling is low — new files compose Manager concepts rather than modify them.

A new package would have required exporting `Resource`, `Readiness*` constants, `fakeRuntimeDriver`, and the Manager internals, creating a worse coupling surface.

## Locking discipline

- `r.mu` (registry RW mutex): protects the `instances` map (lookup, insert, delete).
- `inst.mu` (instance RW mutex): protects individual `ServiceInstance` fields (State, Generation, ContainerID, Endpoint, BoundClients, etc.).
- Lock order: **always** `r.mu` → `inst.mu`. Never reversed to prevent deadlocks.
- Long-running ops (container create/start, readiness probes): release locks before blocking.

## State machine

Valid transitions (implemented in `isLegalTransition()`):

```
DECLARED  → STARTING, STOPPED
STARTING  → READY, FAILED, STOPPING
READY     → UNHEALTHY, FENCED, STOPPING
UNHEALTHY → STARTING, STOPPING
FENCED    → STOPPING
STOPPING  → STOPPED, FAILED
STOPPED   → STARTING
FAILED    → STARTING
```

## Key behaviors implemented

1. **Concurrent start race**: generation CAS; only one caller wins; loser gets `ErrConcurrentStartRace`.
2. **Duplicate start idempotent**: Start on READY with same generation is no-op.
3. **Crash before readiness**: container create/start failure → FAILED.
4. **Crash after readiness**: Reconcile detects → UNHEALTHY.
5. **Stale endpoint**: Reconcile with no live container proof → UNHEALTHY.
6. **Duplicate generation**: two containers same (service, generation) → UNHEALTHY.
7. **Client refcount**: BindClient/UnbindClient; last unbind stops workflow-scoped service.
8. **WorkflowTerminal**: stops all non-STOPPED services for a workflow.
9. **Stop idempotent**: stopping STOPPED service is no-op.
10. **Fence**: only from READY; stores redacted reason.
11. **Promotion check**: Declare validates via `PromotionChecker` interface (fake in tests).
12. **Readiness probe**: injectable via `ReadinessProbe` interface (fake in tests; T05 wires real protocol).
