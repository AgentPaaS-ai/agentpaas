## Implementation decisions

1. **Tool name validation**: Uses `[a-zA-Z][a-zA-Z0-9_.-]*` regex, consistent with policy MCP tool ID conventions.
2. **Max concurrency**: Default 1, range 1–32. Enforced at validation time; the service loop serializes by default. Concurrency semaphore left for T06 (production harness with B30 deadlines).
3. **Runner mode dispatch**: `runner.run()` checks `AGENTPAAS_AGENT_KIND` env var — set only by harness from signed agent.yaml. Never from free-form user payload.
4. **Tool-set equality**: `validate_declared_tools()` performs exact set comparison. Missing declared tool OR extra registered tool both cause `import_failed` with `tool_set_mismatch`.
5. **Embedded runner**: The production path (`python -c pythonRunner`) calls `from agentpaas_sdk import run; run()`, which dispatches to service mode. The legacy fallback (when SDK import fails) does NOT support service mode — acceptable since service packages always include the SDK.
6. **Service loop**: One tool returning does NOT exit the process. Loop continues until EOF or `shutdown` message.
7. **Error bounding**: Tool exceptions mapped to `tool_error` with redacted messages (common secret patterns stripped). No traceback leakage.
8. **Control field rejection**: `__agentpaas_*` keys rejected in both arguments and results. `_FORBIDDEN_RESPONSE_KEYS` (endpoint, host, ip, port, capability_token, etc.) checked on results.
