# Pre-v0.3.0 Manual Test Plan (founder-run via ap-testing)

**Pinned to:** main HEAD `0.3.0-dev` (B26–B32 unreleased)  
**Profile:** `ap-testing` (Hermes)  
**Orchestrator role:** give you paste prompts only — does **not** run the loop  
**Your role:** one phase at a time; on any FAIL → STOP, report, do not workaround  

## Rules (non-negotiable)

1. **Clean slate every major half** (publisher half, receiver half). No leftover daemon, brew agentpaas, or profile plugin.
2. **STOP on bug.** Do not skip. Do not weaken assertions. Fix code later; do not “make the test pass.”
3. **Disk truth only.** PASS requires real `invoke-response.json`, `harness-audit.jsonl`, `bundle inspect`, CLI output — never agent self-report alone.
4. **One paste per phase.** Wait for the phase to finish; verify; then next paste.
5. **Secrets:** you run `agentpaas secret add …` in **your own terminal** (stdin). Never paste API keys into Hermes.
6. **Install vehicle for this RC:** `v0.3.0` is **not** on Homebrew yet.  
   - Golden loop **install** uses a **local RC install** from repo `make build-all` into a disposable prefix (mimics brew layout).  
   - After you approve the tag, re-run Phase 2 once via real `brew install` on the published cask.

## Major claims under test

| ID | Claim | How we prove it |
|----|--------|-----------------|
| C0 | Clean install + doctor | Phase 1–3 |
| C1 | Build/pack/run/invoke weather+LLM | Phase 4 |
| C2 | Modify → new digest, egress confirm | Phase 5–6 |

| C4 | Export signed bundle (9 checks) | Phase 8 |

**Honest scope:** Full multi-agent `agent.delegate(...)` across two live containers is **library/harness-proven in unit tests**, not yet a first-class Hermes “two weather agents talk” golden path. Phase 15 records that as **limitation**, not as PASS for a public A2A claim.
