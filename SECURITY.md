# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AgentPaaS, please report it via
**GitHub Security Advisories** (the "Security" tab → "Report a vulnerability").

**Please do not report security vulnerabilities through public GitHub issues.**

You should receive an initial response within 48 hours.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < latest| :x:                |

Only the latest published version receives security updates.

## Response Timeline

- **Acknowledgment**: within 48 hours of report.
- **Triage / Confirmation**: within 5 business days.
- **Patch / Mitigation**: within 14 days of confirmation (for supported versions).
- **Public Disclosure**: coordinated with reporter; typically 30 days after a fix is released.

## Scope

- **In scope**: the AgentPaaS runtime, CLI, SDK, and official plugins.
- **Out of scope**: third-party dependencies, the landing page, and example/test code.

## Safe Harbor

We participate in responsible disclosure. Researchers who report vulnerabilities in good faith, follow this policy, and allow time for remediation will not face legal action.
