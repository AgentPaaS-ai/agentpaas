# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in AgentPaaS, please report it via email to **contact@agentpaas.ai**.

**Please do not report security vulnerabilities through public GitHub issues.**

You should receive an initial response within 48 hours. If you do not hear back, please follow up to ensure your message was received.

### PGP Key

A PGP key is available for encrypted disclosure if needed. Contact the maintainer to obtain the key.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < latest| :x:                |

AgentPaaS is **pre-release / P1 (Prototype Phase 1) software**. Only the latest published version receives security updates.

## Response Timeline

- **Acknowledgment**: within 48 hours of report.
- **Triage / Confirmation**: within 5 business days.
- **Patch / Mitigation**: within 14 days of confirmation (for supported versions).
- **Public Disclosure**: coordinated with reporter; typically 30 days after a fix is released.

## Scope

- **In scope**: the AgentPaaS runtime, CLI, SDK, and official plugins/skills.
- **Out of scope**: third-party dependencies, the `landing-page/` directory (separate project track), and example/test code.

## Safe Harbor

We participate in responsible disclosure. Researchers who report vulnerabilities in good faith, follow this policy, and allow time for remediation will not face legal action.
